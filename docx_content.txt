Problem Definition
I’m facing a network optimization under constraint problem. The Central Bank may force a 25–50% reduction in operational ATMs. My job is to decide which ATMs to keep running so that:
Service coverage is maximized (geographic, temporal, transactional)
Financial loss is minimized
Risk scenarios are handled (curfew/mall closures, fuel/power shortages)
I have 45 ATMs across Jordan (Amman, Zarqa, Irbid, Mafraq, Karak, Salt, Ramtha, Jerash, Madaba, Aqaba). If I keep 50%, I keep ~22–23 ATMs. If 25%, only ~11.

The Three-Model Approach in Detail
Model 1 — Graph Neural Network (GNN)
Why GNN: ATMs don't exist in isolation. If you remove ATM A, its customers don't disappear — they migrate to the nearest substitute. GNNs model this network dependency natively.
Setup: Build a graph where each ATM is a node with features (transaction volume, services, hours, location, power resilience). Edges connect ATMs within a configurable radius (say 2 km), weighted by inverse distance. You then use a GraphSAGE or Graph Attention Network (GAT) to learn a utility embedding for each node — essentially: "how much network value does this ATM carry?"
The key task you'd train on is coverage loss prediction: given a subset of removed ATMs, predict the number of customers left unserved. With historical data on branch closures (maintenance, outages), you can generate training examples.
Model 2 — Integer Linear Programming (ILP)
Why ILP: Once the GNN gives you a utility score per ATM per scenario, the selection problem is a classic budgeted maximum coverage problem — provably solvable with ILP.
The formulation:
Maximize: Σ (utility_i × x_i)Subject to: Σ x_i ≤ k (where k = 25% or 50% of 45)And: x_i ∈ {0,1} (keep or remove)Scenario constraints: if curfew → x_mall_i = 0 forcedPower constraint: at least 1 generator-backed ATM per region
